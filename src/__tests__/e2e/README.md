# E2E端到端测试说明

## 概述

本目录包含Legal Debate MVP的端到端（E2E）测试套件，用于验证完整的辩论流程。

## 测试目录结构

```
src/__tests__/e2e/
├── debate-flow/
│   ├── helpers.ts              # 测试辅助函数
│   ├── single-round.spec.ts     # 单轮辩论流程测试
│   ├── multi-round.spec.ts      # 多轮辩论流程测试
│   ├── error-handling.spec.ts   # 异常处理测试
│   └── data-consistency.spec.ts # 数据一致性测试
├── performance/
│   ├── response-time.spec.ts    # 响应时间性能测试
│   └── concurrent-users.spec.ts # 并发用户性能测试
└── reports/
    ├── test-reporter.ts        # 测试报告生成器
    └── coverage-checker.ts     # 覆盖率检查器
```

## 测试套件说明

### 1. 单轮辩论完整流程 (single-round.spec.ts)

验证从文档上传到辩论生成的完整流程：

- 创建测试案件
- 上传测试文档
- 文档解析完成验证
- 法条检索
- 法条适用性分析
- 创建辩论
- 生成辩论论点
- 数据一致性验证

### 2. 多轮辩论流程 (multi-round.spec.ts)

验证多轮辩论的上下文继承和论点递进：

- 两轮辩论流程测试
- 上下文继承验证
- 论点递进验证
- 增量分析验证
- 三轮辩论完整支持验证

### 3. 异常处理流程 (error-handling.spec.ts)

验证各种异常情况的处理机制：

- 文档解析失败处理
- 法条检索无结果处理
- AI服务超时处理
- AI服务错误处理
- SSE连接中断处理
- 并发请求冲突处理
- 系统状态恢复验证

### 4. 数据一致性测试 (data-consistency.spec.ts)

验证跨模块数据传递和状态同步：

- 文档解析结果与案件关联
- 法条检索结果与辩论引用
- 辩论轮次与论点关联
- 缓存数据与数据库一致性
- AI交互记录完整性
- 用户操作日志完整性

### 5. 响应时间性能测试 (response-time.spec.ts)

验证关键接口的响应时间：

- 文档上传 < 3秒
- 文档解析 < 20秒
- 法条检索 < 1秒
- 适用性分析 < 2秒
- 辩论生成 < 30秒
- 完整流程 < 60秒
- 数据库查询 < 100ms

### 6. 并发用户性能测试 (concurrent-users.spec.ts)

验证系统在多用户同时访问时的稳定性：

- 并发创建案件（10用户）
- 并发查询案件（10用户）
- 并发创建辩论（5用户）
- 并发混合操作（10用户）
- 高并发场景（50请求）
- 并发冲突处理
- 持续并发压力测试

## 运行测试

### 前置条件

1. 确保开发服务器正在运行：

   ```bash
   npm run dev
   ```

2. 确保数据库连接正常

### 运行所有E2E测试

```bash
npm run test:e2e
```


## 常见问题

### Q1: 测试超时怎么办？

A: 检查开发服务器是否正常运行，数据库连接是否稳定。可以增加超时时间。

### Q2: 部分测试失败怎么办？

A: 首先检查测试数据是否准备充分，API接口是否正常。查看测试报告获取详细错误信息。

### Q3: 性能测试不达标怎么办？

A: 性能测试受环境影响较大，建议多次运行取平均值。如果持续不达标，需要优化代码性能。

### Q4: 如何调试测试？

A: 使用Playwright的debug模式运行测试，可以逐步查看浏览器状态：

```bash
npm run test:e2e -- --debug
```

## 测试报告解读

### 通过率

- **100%**：所有测试通过，系统质量优秀
- **90-99%**：大部分测试通过，存在少量问题
- **<90%**：存在较多问题，需要重点关注

### 覆盖率

- **>90%**：E2E流程覆盖充分
- **80-90%**：基本覆盖，可以补充测试
- **<80%**：覆盖不足，需要大量补充

### 性能指标

- **达标率 >90%**：性能表现优秀
- **达标率 80-90%**：性能良好，有优化空间
- **达标率 <80%**：性能需要优化
