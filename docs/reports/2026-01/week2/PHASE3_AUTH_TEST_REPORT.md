## 发现的问题和改进建议

### 问题1: 测试数据清理机制不完善

**描述**: 测试中依赖 `/api/auth/test/cleanup/${userId}` 端点，但该端点可能不存在。

**影响**: 测试后可能残留测试数据，导致后续测试失败。

**建议优先级**: 高

**改进方案**:

```typescript
// 方案1: 创建测试清理API端点
// src/app/api/auth/test/cleanup/[userId]/route.ts
export async function DELETE(
  request: NextRequest,
  { params }: { params: { userId: string } }
) {
  // 仅在测试环境执行
  if (process.env.NODE_ENV !== 'test') {
    return NextResponse.json({ error: 'Forbidden' }, { status: 403 });
  }

  // 清理用户及相关数据
  await prisma.user.delete({
    where: { id: params.userId },
  });

  return NextResponse.json({ success: true });
}

// 方案2: 在测试中直接使用Prisma清理
async function cleanupTestUser(userId: string) {
  await prisma.verificationCode.deleteMany({ where: { userId } });
  await prisma.session.deleteMany({ where: { userId } });
  await prisma.lawyerQualification.deleteMany({ where: { userId } });
  await prisma.enterpriseAccount.deleteMany({ where: { userId } });
  await prisma.user.delete({ where: { id: userId } });
}
```
