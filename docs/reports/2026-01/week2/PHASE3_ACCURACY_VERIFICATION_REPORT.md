## 3. 实施内容

### 3.1 测试数据集管理工具

创建了`scripts/accuracy-test-data-manager.ts`，功能包括：

1. **从数据库导出文档**
   - 支持指定数量限制
   - 自动生成模拟文档内容
   - 从现有提取数据构建黄金标准

2. **创建多样化测试文档**
   - 支持5种案件类型
   - 每种类型3-15个变体
   - 自动生成标准格式的法律文档

3. **AI辅助标注**
   - 使用AI自动提取文档信息
   - 生成JSON格式的黄金标准
   - 支持批量标注

4. **统计查询**
   - 测试集文档总数
   - 按案件类型分布
   - 已标注文档数量

### 3.2 批量准确性测试脚本

创建了`scripts/run-accuracy-batch-test.ts`，功能包括：

1. **批量测试执行**
   - 支持50个文档的批量测试
   - 逐个文档执行DocAnalyzer分析
   - 使用AI验证提取结果

2. **多维度评估**
   - 当事人识别准确率
   - 诉讼请求提取准确率
   - 完整性、一致性、清晰度评估
   - AI验证置信度

3. **按案件类型分组**
   - 分别统计各类型准确率
   - 识别类型间准确率差异
   - 针对性优化建议

4. **详细报告生成**
   - 总体准确率统计
   - 各维度详细结果

## 6. 使用说明

### 6.1 创建测试数据集

```bash
# 创建50个多样化测试文档
npx tsx scripts/accuracy-test-data-manager.ts create

# 查看测试集统计
npx tsx scripts/accuracy-test-data-manager.ts stats

# 从数据库导出文档（可选）
npx tsx scripts/accuracy-test-data-manager.ts export 50
```

### 6.2 运行准确性测试

```bash
# 测试全部50个文档
npx tsx scripts/run-accuracy-batch-test.ts 50

# 测试指定数量文档
npx tsx scripts/run-accuracy-batch-test.ts 10
```

### 6.3 查看测试报告

测试完成后，报告将自动保存至：

- 文件路径：`test-results/accuracy-batch-report.json`

报告内容包括：

- 总体准确率统计
- 按案件类型分组结果
- 每个文档的详细结果
- 改进建议
