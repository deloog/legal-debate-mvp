**模式数量**: 8个

#### 3.4 替代关系模式 (SUPERSEDE_PATTERNS)

支持3种废止模式：
- 同时废止、自...施行之日起废止、原...废止

**模式数量**: 3个

支持3种补充模式：
- 对...的补充规定、补充...、...的补充

**模式数量**: 3个

### 4. 规则检测器 (rule-based-detector.ts)

实现了RuleBasedDetector类，包含5个静态方法：

#### 4.1 detectCitesRelation

**功能**: 检测引用关系

**输入**: LawArticle对象

**输出**: CitesRelation数组

**算法**:
1. 遍历所有引用模式
2. 使用正则表达式匹配法条全文
3. 提取法律名称和条款号
4. 返回置信度0.95的关系列表

**特点**:
- 支持多个引用关系检测
- 自动处理书名号和无书名号情况
- 重置正则表达式lastIndex避免状态污染

#### 4.2 detectHierarchicalRelation

**功能**: 检测层级关系（上位法/下位法）

**输入**: LawArticle对象

**输出**: HierarchicalRelation数组

**算法**:
1. 遍历所有层级模式
2. 匹配"根据...制定"等模式
3. 提取上位法名称
4. 返回置信度0.85的IMPLEMENTS关系

**特点**:
- 识别实施细则与上位法关系
- 支持多种制定关键词

#### 4.3 detectConflictsRelation

**功能**: 检测冲突关系

**输入**: LawArticle对象

**输出**: ConflictRelation数组

**算法**:
1. 遍历所有冲突模式
2. 匹配"与...不一致"等模式
3. 提取冲突法律名称
4. 返回置信度0.8的冲突关系

**特点**:
- 识别法条间的矛盾和冲突
- 支持多种冲突表述

#### 4.4 detectSupersedesRelation

**功能**: 检测替代关系（新法替旧法）

**输入**: LawArticle数组

**输出**: SupersedesRelation数组

**算法**:
1. 按lawName分组法条
2. 按effectiveDate排序
3. 相邻版本建立SUPERSEDES关系
4. 检测文本中的废止声明
5. 返回置信度0.9-0.95的替代关系

**特点**:
- 自动识别同名法律的版本替代
- 支持文本中的明确废止声明
- 过滤无效日期的法条

#### 4.5 detectAllRelations

**功能**: 批量检测所有关系

**输入**: LawArticle对象

**输出**: DetectionResult对象（包含cites、hierarchical、conflicts）

**算法**:
1. 并行调用前3个检测方法
2. 汇总所有检测结果
3. 返回统一的结果对象

**特点**:
- 一次调用获取所有关系
- 异步方法支持未来扩展

**代码行数**: 约200行

## 📝 使用示例

### 检测引用关系

```typescript
import { RuleBasedDetector } from '@/lib/law-article/relation-discovery/rule-based-detector';

const article = {
  fullText: '根据《民法典》第10条的规定，当事人应当遵守诚实信用原则。',
  // ... 其他字段
};

const cites = RuleBasedDetector.detectCitesRelation(article);
// 返回: [{ lawName: '民法典', articleNumber: '10', evidence: '根据《民法典》第10条', confidence: 0.95 }]
```

### 检测层级关系

```typescript
const article = {
  fullText: '根据《中华人民共和国宪法》制定本法。',
  // ... 其他字段
};

const hierarchical = RuleBasedDetector.detectHierarchicalRelation(article);
// 返回: [{ parentLawName: '中华人民共和国宪法', relationType: 'IMPLEMENTS', confidence: 0.85 }]
```

### 批量检测所有关系

```typescript
const result = await RuleBasedDetector.detectAllRelations(article);
// 返回: { cites: [...], hierarchical: [...], conflicts: [...] }
```

## 🚀 性能特点

### 时间复杂度

- **单次检测**: O(n*m)，n为文本长度，m为模式数量
- **批量检测**: O(k*n*m)，k为法条数量
- **版本排序**: O(k*log(k))，k为同名法律版本数

### 空间复杂度

- **内存占用**: O(r)，r为检测到的关系数量
- **临时存储**: O(k)，k为分组后的法律数量

### 实际性能

- **单法条检测**: <10ms
- **100次重复文本**: <50ms
- **批量检测**: 线性增长
