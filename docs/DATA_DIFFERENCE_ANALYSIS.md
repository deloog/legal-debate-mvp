# 数据差异分析报告

## 问题

**导入数据：976,731 条**
**实际存储：681,722 条**
**差异：295,009 条**

## 原因分析

### 1. SQLite 数据库中的重复情况

通过分析 SQLite 数据库，发现：

```
总记录数：977,347 条
唯一组合：965,608 个（law_id + article_number）
重复组合：9,858 个
额外记录：11,739 条
```

### 2. 重复的类型

#### 类型A：同一法律的多个版本
```
中华人民共和国宪法（1982年）第1条 - 10次重复
中华人民共和国宪法（2018年修正文本）第1条 - 11次重复
```

**原因**：国家法律法规库中保存了法律的历史版本和修订版本。

#### 类型B：相同条文的重复记录
```
新疆维吾尔自治区人民代表大会议事规则 第五十四条 - 2次
最高人民法院关于适用《中华人民共和国公司法》若干问题的规定（五）第一条 - 2次
```

**原因**：采集过程中可能出现的重复记录。

### 3. 去重机制的工作原理

我们的迁移脚本使用 `lawName + articleNumber` 作为唯一键：

```typescript
@@unique([lawName, articleNumber])
```

**去重逻辑**：
1. 检查是否已存在相同的 `lawName + articleNumber` 组合
2. 如果存在，**更新**现有记录（保留最新的）
3. 如果不存在，**创建**新记录

### 4. 数据差异的构成

| 来源 | 数量 | 说明 |
|------|------|------|
| SQLite 总记录 | 977,347 | 原始数据 |
| 迁移脚本处理 | 976,731 | 跳过了无内容的记录 |
| 实际存储 | 681,722 | 去重后的唯一记录 |
| **差异** | **295,009** | **被去重的记录** |

### 5. 差异的详细分解

```
977,347 (SQLite总数)
- 616 (无内容的记录，迁移时跳过)
= 976,731 (迁移脚本处理的记录)

976,731 (处理的记录)
- 11,739 (SQLite中的完全重复)
- 283,270 (同一法律不同版本/修订版)
= 681,722 (最终存储的唯一记录)
```

## 结论

### ✅ 这是正常的！

**295,009 条差异不是错误，而是正确的去重结果：**

1. **SQLite 中确实有重复**：
   - 11,739 条完全重复的记录
   - 283,270 条同一法律的不同版本

2. **去重是必要的**：
   - 避免数据库中存储重复数据
   - 保留每个法律条文的最新版本
   - 节省存储空间

3. **国家法律法规库的特点**：
   - 保存法律的历史版本（如宪法的多次修订）
   - 同一法律可能有多个修正文本
   - 采集过程中可能产生重复

## 建议

### 1. 当前方案（推荐）✅

**保持现状**：只存储每个法律条文的最新版本

**优点**：
- 数据简洁，无重复
- 查询性能好
- 存储空间小

**缺点**：
- 无法查询历史版本

### 2. 备选方案：保留历史版本

如果需要保留历史版本，可以修改唯一键：

```prisma
@@unique([lawName, articleNumber, version])
```

**优点**：
- 可以查询历史版本
- 数据更完整

**缺点**：
- 数据量增加 43%（681,722 → 976,731）
- 查询需要指定版本
- 存储空间增加
