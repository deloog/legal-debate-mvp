2. 实现错误提示中间件
3. 错误提示文案：
   - 超时："分析时间较长，请耐心等待"
   - 失败："当前服务繁忙，请稍后重试"

**技术实现**：

```typescript
// src/lib/middleware/ai-error-handler.ts
export class AIServiceErrorHandler {
  private static readonly TIMEOUT_MS = 30000; // 30秒

  static async withTimeout<T>(
    promise: Promise<T>,
    context: string,
  ): Promise<T> {
    const timeoutPromise = new Promise<never>((_, reject) => {
      setTimeout(() => reject(new Error("AI服务调用超时")), this.TIMEOUT_MS);
    });

    try {
      return await Promise.race([promise, timeoutPromise]);
    } catch (error) {
      if (error.message === "AI服务调用超时") {
        throw new AIServiceTimeoutError(context);
      }
      throw new AIServiceFailureError(context, error);
    }
  }
}
```

**实施架构**：**"AI识别+算法兜底+AI审查"五层架构**

##### 五层处理架构

基于项目实施经验，文档解析系统采用以下五层复合架构：

- **Layer 0: TextExtractor（文本提取层）**
  - 职责：PDF/Word/TXT文本提取
  - 技术：pdf-parse、mammoth等库
  - 输出：纯文本内容

- **Layer 1: FilterProcessor（预处理和分类层）**
  - 职责：OCR质量检查、文档分类、格式验证
  - 技术：规则过滤、格式检测
  - 输出：预处理后的文本和元数据

- **Layer 2: AIProcessor（AI识别层）**
  - 职责：当事人角色识别、诉讼请求分类、金额模糊识别
  - 技术：智谱清言/DeepSeek多服务商支持
  - 配置：温度0.1，最大Token 4000
  - 输出：AI识别的结构化数据

- **Layer 3: RuleProcessor（算法兜底层）**
  - 职责：ClaimExtractor（诉讼请求补充）、AmountExtractor（金额优化）
  - 技术：规则匹配、正则表达式、语义扩展
  - 策略：补充AI遗漏、修正错误、拆解复合请求
  - 输出：规则增强的解析结果

- **Layer 4: ReviewerManager（AI审查层）**
  - 职责：AIReviewer（二次审查）+ RuleReviewer（规则审查）
  - 技术：AI审查 + 规则验证
  - 内容：当事人准确性、诉讼请求完整性、数据一致性、金额准确性、法律依据合理性
  - 设计：corrections为建议性质，不自动应用
  - 输出：质量审查报告

- **Layer 5: CacheProcessor（缓存优化层）**
  - 职责：Redis缓存、性能优化
  - 技术：SHA256哈希、TTL控制
  - 策略：避免重复计算、提升响应速度
  - 输出：缓存命中的解析结果

##### 完整数据流验证

```
DocAnalyzerAgent.executeLogic()
  → TextExtractor.extractText()
  → FilterProcessor.process()
  → AIProcessor.process() → DocumentParser.analyzeWithAI() → 真实AI服务
  → RuleProcessor.process() → ClaimExtractor + AmountExtractor（算法兜底）
  → LegalRepresentativeFilter.applyToExtractedData()
  → ReviewerManager.review() → AIReviewer.review() → 真实AI审查
  → CacheProcessor.set()
  → 返回 DocumentAnalysisOutput
```

## 🔄 Sprint 3：辩论核心系统（3周）⭐ 特色功能

### 业务目标

实现完整的辩论流程：

```
用户录入案情
    ↓
AI分析案情（DocAnalyzer - 五层架构）
    ↓
识别提炼辩论所需法条
    ↓
本地法条库检索 → 未找到/不全面 → 法律之星外部检索
    ↓ 找到          ↓
    ←───────────────┘
    ↓
法条适用性分析
    ↓
AI辩论生成（第一轮）
    ↓
用户提交新证据/法条
    ↓
开启新轮次辩论（第二、三轮...）
```

### 设计原则

1. **先本地后外部**：优先使用本地法条库，降低成本提高速度
2. **五层架构复用**：借鉴DocAnalyzer的成功经验（AI识别+算法兜底+AI审查）
3. **渐进式交付**：分阶段实现检索、分析、辩论功能
4. **多轮辩论支持**：允许用户随时补充证据和法条
