## 阶段一：核心Agent集成（第1-3周）

> **目标**：将 VerificationAgent 的真实计算结果接入辩论流程，替换固定分数
> **优先级**：🔴 P0

### 1.1 VerificationAgent 动态评分集成

**任务编号**: P0-001
**工期**: 5天

#### 问题根因

当前辩论生成 API（`/api/v1/debate-rounds/[roundId]/generate/route.ts`）中，`generateSideArguments` 函数对每条论点写入的是硬编码分数：

```typescript
// 当前（固定值，非 Agent 计算）
logicScore: 0.85,
legalScore: 0.9,
overallScore: 0.87,
```

VerificationAgent 已实现三重验证（事实准确性 40% + 逻辑一致性 30% + 任务完成度 30%），但从未被辩论流程调用。

#### 工作内容

| 子任务                             | 工时  | 说明                                                           |
| ---------------------------------- | ----- | -------------------------------------------------------------- |
| 在辩论生成后调用 VerificationAgent | 2天   | 生成论点后异步调用 `verificationAgent.verify()`                |
| 将验证结果回写到 Argument 表       | 1天   | 更新 `legalScore`/`logicScore`/`overallScore`                  |
| 添加验证详情弹窗组件               | 1.5天 | 点击分数展示三重验证明细                                       |
| 新增验证 API 路由                  | 0.5天 | `GET /api/v1/debates/[id]/arguments/[argumentId]/verification` |

#### 技术实现

```typescript
// src/app/api/v1/debate-rounds/[roundId]/generate/route.ts
// 修改：生成论点后异步验证并回写分数
import { VerificationAgent } from '@/lib/agent/verification-agent';

// 生成论点保存后，异步调用验证（不阻塞主流程）
void (async () => {
  const agent = new VerificationAgent();
  for (const arg of allArguments) {
    const result = await agent.verify({ content: arg.content, ... });
    await prisma.argument.update({
      where: { id: arg.id },
      data: {
        legalScore: result.scores.factual,
        logicScore: result.scores.logical,
        overallScore: result.scores.overall,
      },
    });
  }
})();

// 新增组件
src/components/verification/
├── VerificationDetailModal.tsx   # 三重验证详情弹窗
└── VerificationBadge.tsx         # 验证徽章（通过/警告/失败）
```

#### 验收标准

- [x] 辩论完成后，论点分数由 VerificationAgent 动态计算（非固定值）
- [x] 前端展示事实准确性、逻辑一致性、任务完成度三项分数
- [x] 验证在后台异步执行，不阻塞辩论生成响应
- [x] 验证失败时降级为原有固定分数

#### 实现状态: ✅ 已完成 (2026-03-31)

**实际实现详情**:

- **核心服务**: `src/lib/debate/argument-verification-service.ts` - 并行验证 + 批量更新
- **集成点**: `src/lib/debate/debate-generator.ts` - 生成后自动调用验证
- **前端组件**: `src/components/verification/VerificationDetailModal.tsx` - 三重验证明细展示
- **API 路由**: `GET /api/v1/arguments/[id]/verification` - 获取验证详情
- **数据库**: `Argument.metadata` JSON 字段存储完整验证数据

**性能优化**: 并行验证实现，理论性能提升 ~80% (串行 3s → 并行 0.5s)

**审计报告**: 详见 `docs/P0-001_VERIFICATION_AUDIT.md` (综合评分: A/93%)
