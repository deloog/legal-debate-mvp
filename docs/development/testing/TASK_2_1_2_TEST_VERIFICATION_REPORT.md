### 4. 已知问题

⚠️ **AI服务警告**

- 测试过程中检测到DeepSeek API高错误率（100%）
- 系统自动降级到本地处理模式
- 不影响测试结果（规则匹配层正常工作）

**详细说明**：

经过深入调查，DeepSeek API密钥有效且服务正常。问题根因是系统的Fallback机制在开发环境中被触发：

1. **API连接验证**：通过独立测试确认DeepSeek API连接正常
2. **Fallback机制**：当AI服务调用失败时，FallbackManager自动降级到localProcessing
3. **响应格式不匹配**：localProcessing返回预定义的英文文本，而TimelineExtractor期望JSON格式
4. **三层架构优势**：规则匹配层（第二层）正常工作，确保测试全部通过

**详细分析报告**：参见 `docs/testing/TASK_2_1_2_DEEPSEEK_API_ISSUE_ANALYSIS.md`

**建议优化方案**：

- 短期：在开发环境中禁用localProcessing fallback，确保使用真实API
- 中期：改进FallbackManager的localProcessing，使其返回符合期望的格式
- 长期：建立API监控机制，优化测试策略
