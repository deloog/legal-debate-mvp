# 真实法律数据获取指南

## 重要说明

⚠️ **警告**：您data目录下的JSON文件是**开发测试用的种子数据**，不是真实完整的法律数据。

### 种子数据的问题

1. **不完整**：只包含少数几条法条作为示例
2. **不准确**：可能包含错误或过期的内容
3. **过时**：没有及时更新到最新版本
4. **需要清除**：在导入真实数据前应该删除

### 关于"民事合同法"

❌ **不存在独立的《中华人民共和国合同法》**：
- 1999年的《合同法》已于2021年1月1日废止
- 现在所有合同法律都在《中华人民共和国民法典》第三编"合同编"中
- 正确的法律名称应该是"中华人民共和国民法典"

## 获取真实法律数据的途径

### 1. 司法部官网（推荐，免费）

**地址**：https://flk.npc.gov.cn/

**特点**：
- ✅ 官方权威来源
- ✅ 完全免费
- ✅ 数据完整准确
- ✅ 实时更新
- ⚠️ 需要手动下载和转换

**下载步骤**：
1. 访问 https://flk.npc.gov.cn/
2. 在搜索框中搜索需要的法律
3. 找到后点击进入详情页
4. 点击"下载"或"导出"按钮
5. 选择合适的格式（建议JSON或XML）
6. 保存到本地

**需要下载的主要法律**：
- 中华人民共和国民法典
- 中华人民共和国刑法
- 中华人民共和国刑事诉讼法
- 中华人民共和国民事诉讼法
- 中华人民共和国行政诉讼法
- 其他常用法律法规

**数据格式转换**：
司法部下载的数据可能需要转换为系统要求的格式：
```json
{
  "data": [
    {
      "lawName": "中华人民共和国民法典",
      "articleNumber": "第一条",
      "fullText": "法条完整内容",
      "lawType": "LAW",
      "category": "CIVIL",
      "effectiveDate": "2021-01-01"
    }
  ]
}
```

**获取方式**：

3. 通过API获取案例数据
4. 转换为系统要求的格式

**获取方式**：

```bash
git clone https://github.com/AndrewZhe/law-gpt.git

cd law-gpt
```

## 清除种子数据

在导入真实数据之前，建议先清除种子数据。

### 方式一：使用命令行（推荐）

```bash
# 清除所有法条数据
npm run clear:law-articles

# 清除指定数据源的数据
npm run clear:law-articles local
```

### 方式二：使用管理后台

1. 登录管理后台
2. 进入 `法条管理` 页面
3. 勾选需要删除的法条
4. 点击 `批量删除` 按钮
5. 确认删除

## 导入真实数据

### 准备工作

1. **下载真实法律数据**：从上述任一途径获取
2. **转换数据格式**：转换为系统要求的JSON格式
3. **验证数据**：检查必填字段是否完整

### 数据格式要求

#### 最小字段（必填）

```json
{
  "data": [
    {
      "lawName": "法律名称",
      "articleNumber": "条号",
      "fullText": "法条完整内容",
      "lawType": "LAW",
      "category": "CIVIL",
      "effectiveDate": "2021-01-01"
    }
  ]
}
```

#### 完整字段（推荐）

```json
{
  "data": [
    {
      "lawName": "法律名称",
      "articleNumber": "条号",
      "fullText": "法条完整内容",
      "lawType": "LAW",
      "category": "CIVIL",
      "subCategory": "子类别",
      "tags": ["标签1", "标签2"],
      "keywords": ["关键词1", "关键词2"],
      "version": "1.0",
      "effectiveDate": "2021-01-01",
      "expiryDate": "2025-01-01",
      "status": "VALID",
      "issuingAuthority": "颁布机关",
      "jurisdiction": "适用地区",
      "searchableText": "可搜索文本"
    }
  ]
}
```

### 导入方法

#### 方式一：管理后台导入（推荐）

1. **清除种子数据**（如果需要）
   ```bash
   npm run clear:law-articles
   ```

2. **登录管理后台**
   - 访问：`http://localhost:3000/admin/law-articles`

3. **导入数据**
   - 点击 `导入法条` 按钮
   - 选择数据源：根据实际来源选择（如"司法部"、"CAIL"等）
   - 选择导入方式：`上传文件`
   - 选择准备好的JSON文件
   - 点击 `导入`

4. **查看结果**
   - 导入完成后会显示成功/失败数量
   - 刷新列表查看导入的数据

#### 方式二：API导入（高级用户）

```bash
curl -X POST http://localhost:3000/api/admin/law-articles/import \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d @your-laws.json
```

## 数据源策略

### 查询优先级（三级缓存）

1. **内存缓存**：最快，命中率最高
2. **本地数据库**：司法部、CAIL等开源数据
3. **外部API**：法律之星、北大法宝等商业API

### 外部API使用策略

- 外部API作为补充，用于本地数据库没有的数据
- 每次调用外部API成功后，将结果持久化到数据库
- 支持配额监控，接近上限时自动告警
- 配额用尽时自动降级到本地数据库

## 推荐的数据获取方案

### 方案一：司法部 + 本地存储（推荐，免费）

**优势**：
- ✅ 完全免费
- ✅ 数据权威准确
- ✅ 无配额限制
- ✅ 可持续更新

**步骤**：
1. 从司法部官网下载主要法律
2. 转换为标准格式
3. 导入到系统数据库
4. 定期检查更新并导入新法条

**适用场景**：个人使用、小团队

### 方案二：专业数据库API（付费，最完整）

**优势**：
- ✅ 数据最全面
- ✅ 自动更新
- ✅ 无需手动维护
- ❌ 需要付费订阅

**步骤**：
1. 购买法律之星或北大法宝API服务
2. 获取API密钥
3. 配置到系统环境变量
4. 系统自动调用API获取数据

**适用场景**：商业使用、大型团队

### 方案三：混合方案（平衡）

**优势**：
- ✅ 常用法律用本地数据（快速）
- ✅ 罕见法律用API查询（完整）
- ✅ 成本可控

**步骤**：
1. 从司法部导入常用法律到本地
2. 购买专业数据库API服务
3. 系统优先查询本地数据
4. 本地没有时调用API
5. API返回的结果自动缓存到本地

**适用场景**：中型团队、需要平衡成本和完整性

## 常见问题

### Q: 种子数据一定要清除吗？

A: 强烈建议清除。种子数据不完整不准确，可能会误导系统。

### Q: 从司法部下载的数据格式不对怎么办？

A: 使用提供的转换脚本或手动转换为标准JSON格式。

### Q: 可以混合使用多个数据源吗？

A: 可以。系统支持多数据源，不同数据源的数据可以共存。

### Q: 导入后如何验证数据是否正确？

A: 
1. 在法条管理页面查看数量
2. 搜索测试法条是否正常显示
3. 检查法条内容是否完整准确

### Q: 如何定期更新法律数据？

A: 
1. 定期访问司法部官网检查更新
2. 下载新修订的法条
3. 通过管理后台导入
4. 系统会自动更新已存在的法条

### Q: 外部API配额用完了怎么办？

A: 
1. 系统会自动降级到本地数据库
2. 可以购买更多配额或升级套餐
3. 或者从其他免费渠道获取数据
