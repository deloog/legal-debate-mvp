# AI助手协作规范

> **目标读者**：AI助手、开发团队  
> **适用范围**：多个AI同时参与项目开发时  
> **最后更新**：2026-02-23

---

## 📋 概述

本文档定义了多个AI助手同时参与项目开发时的协作规范，旨在避免重复工作、减少冲突、提升开发效率。

---

## 🎯 基本原则

### 1. 单一职责

每个AI助手应该有明确的任务范围：

- **任务分配原则**：每个AI助手专注于一个模块或功能
- **避免交叉**：不同AI尽量不修改同一文件
- **明确边界**：在开始任务前确认任务范围

### 2. 信息共享

- **任务追踪**：所有AI助手应通过 `docs/task-tracking/AI_TASK_TRACKING.md` 了解当前进度
- **状态更新**：完成任务后及时更新任务状态
- **问题记录**：遇到问题或发现已存在的问题，记录到相应文档

### 3. 避免重复

- **先查后做**：在开始新任务前，先搜索现有实现
- **复用优先**：优先在现有代码基础上扩展，而非创建新文件
- **禁止复制**：禁止创建重复文件（增强版、v2等）

---

## 📝 任务分配流程

### 步骤1：了解当前状态

在开始任何任务前，AI助手应该：

1. 阅读 [`AI_TASK_TRACKING.md`](../../task-tracking/AI_TASK_TRACKING.md) 了解当前任务列表
2. 查看已标记为"待处理"或"进行中"的任务
3. 确认没有其他AI正在处理相同或相关的任务

### 步骤2：声明任务

当AI助手开始一个新任务时：

```markdown
# 在 AI_TASK_TRACKING.md 中添加

[ ] 任务描述 🕐 2026-02-23 (AI助手名称 进行中)
```

### 步骤3：执行任务

按照 `.clinerules` 和相关开发指南执行任务：

1. 遵循代码规范
2. 编写测试用例
3. 更新相关文档

### 步骤4：完成任务

完成任务后，更新任务状态：

```markdown
[x] 任务描述 ✅ 2026-02-23 (AI助手名称 完成)
```

## ⚠️ 冲突处理

### 情况1：两个AI同时处理同一文件

**解决流程**：

1. **先发现者优先**：先开始工作的AI继续处理
2. **沟通协调**：通过任务追踪文档进行协调
3. **避免覆盖**：后开始的AI应选择其他任务或等待

### 情况2：发现其他AI引入的问题

**处理方式**：

1. **记录问题**：在 `docs/archive/problems-and-solutions.md` 中记录
2. **不擅自修改**：不修改其他AI负责的代码（除非紧急修复）
3. **通知负责AI**：在任务追踪中标记问题

### 情况3：任务范围重叠

**解决方式**：

1. **明确边界**：重新定义任务范围
2. **协商分工**：AI之间通过任务追踪文档协调
3. **优先现有实现**：优先使用现有实现，避免重复开发

## ✅ 任务完成检查清单

在报告任务完成时，确保满足以下条件：

- [ ] 已在 `AI_TASK_TRACKING.md` 中更新任务状态
- [ ] 相关测试通过
- [ ] 代码符合 `.clinerules` 规范
- [ ] 已更新相关文档（如需要）
- [ ] 没有引入新的ESLint错误
- [ ] 没有破坏现有功能

## 📞 紧急情况处理

### 紧急修复

当发现影响生产的紧急问题时：

1. **立即通知**：在任务追踪中标记为紧急
2. **快速修复**：优先处理，不拘泥于常规流程
3. **事后记录**：修复后记录问题原因和解决方案

### 系统故障

当遇到系统性问题（如数据库故障、API不可用等）：

1. **停止工作**：不继续可能受影响的开发任务
2. **记录状态**：在任务追踪中标记当前状态
3. **等待恢复**：等待系统恢复后继续工作

---

## 🎓 最佳实践

### 推荐做法

1. **先读后写**：先阅读现有代码和文档，再开始开发
2. **小步快跑**：每次修改尽量小，便于审查和回滚
3. **及时沟通**：通过任务追踪文档与其他AI保持同步
4. **记录决策**：重要的技术决策记录到ADR

### 避免做法

1. **避免重复**：不创建已有功能的重复实现
2. **避免冲突**：不同时修改同一文件
3. **避免猜测**：不确定时先查文档或询问
4. **避免沉默**：不在任务追踪中"失踪"
